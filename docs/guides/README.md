# Guides

Step-by-step walkthroughs and how-to guides for working with the 10-agent workflow.

## Available Guides

- **[Getting Started](getting-started.md)** - 30-second overview and your first feature walkthrough

## How to Use These Guides

### For Absolute Beginners

1. **First:** Read [Getting Started](getting-started.md) (5-10 minutes)
   - Learn the 30-second mental model
   - Understand the 5 categories of agents
   - See your first feature flow

2. **Second:** Walk through [Workflow Example](../WORKFLOW-EXAMPLE.md) (10 minutes)
   - See a complete real-world example with actual commands
   - Understand what each agent produces
   - Learn how artifacts flow together

3. **Third:** Keep these references open while working:
   - [Commands Reference](../reference/commands.md) - What each command does
   - [Workflows](../workflows/README.md) - Decision rules for your specific situation
   - [Terminology](../reference/terminology.md) - Definitions of key terms

### For Setting Up a New Repository

Follow this order:

1. Open [Getting Started](getting-started.md) → "New Repository Setup" section
2. Run `/constitution` to establish durable repo rules
3. Run `/project-knowledge-base` to document architecture and patterns
4. You're ready for feature work!

See [Foundation & Setup documentation](../agents/details/foundation-setup.md) for detailed guidance.

### For Building Your First Feature

Follow this order:

1. Read [Getting Started](getting-started.md) → "Your First Feature" section
2. Decide which workflow to use from [Workflows](../workflows/README.md)
3. Execute each step and review the output
4. Reference [Commands Reference](../reference/commands.md) for details on each command

See [Workflow Example](../WORKFLOW-EXAMPLE.md) for a complete walkthrough.
